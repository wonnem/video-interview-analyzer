# 🎤 AI 영어 면접 챗봇 STT 파이프라인 (MVP)

## 1. 기능 개요
지원자의 영어 면접 응시 영상 데이터에서 음성(Audio)만 추출한 뒤, 이를 자연어 텍스트(Text)로 변환하여 후속 평가 엔진 및 화면에 전달하는 핵심 데이터 전처리 파이프라인입니다.

## 2. 사용 모델 및 기술 스택 (도입 배경)

### 1) 동영상 → 음성 추출: **FFmpeg (via Python `moviepy` 또는 `pydub`)**
* **사용 이유:** * **압도적인 처리 속도와 안정성:** FFmpeg는 미디어 처리 분야의 전 세계 표준입니다. 동영상 파일(.mp4, .webm 등)에서 영상 트랙을 날리고 오디오 트랙(.wav, .mp3)만 분리하는 작업을 몇 줄의 코드만으로 가장 빠르고 가볍게 처리할 수 있습니다.
    * **MVP 적합성:** 복잡한 영상 처리 AI를 도입할 필요 없이, 서버 리소스를 최소화하며 즉각적인 추출이 가능합니다.

### 2) 음성 → 텍스트(STT) 변환: **OpenAI Whisper API (또는 `faster-whisper`)**
* **사용 이유:**
    * **최상위권 영어 인식 정확도:** 이번 과제의 핵심은 지원자의 영어 실력(문법, 표현, 유창성)을 CEFR 기준으로 정확히 평가하는 것입니다. Whisper 모델은 다양한 억양과 발음 뭉개짐 속에서도 문맥을 파악해 정확하게 텍스트로 변환하는 능력이 탁월합니다.
    * **개발 시간 단축 (2주 스프린트 최적화):** 자체 STT 모델을 파인튜닝할 시간이 없는 MVP 단계에서는, API 호출 한 번으로 고품질의 텍스트 결과물(타임스탬프 포함)을 반환해 주는 Whisper API가 가장 효율적인 선택입니다. (비용이나 보안이 문제라면 로컬에서 가볍게 도는 `faster-whisper` 패키지로 대체 가능)
    * **노이즈 캔슬링 효과:** 면접 환경에서 발생할 수 있는 약간의 백그라운드 소음을 자체적으로 어느 정도 필터링하여 텍스트를 추출해 줍니다.

## 3. 데이터 처리 플로우

1.  **Input:** 클라이언트(웹/앱)로부터 지원자의 답변 동영상 파일 수신 (ex: `answer_01.mp4`)
2.  **Audio Extraction:** FFmpeg를 활용해 동영상에서 16kHz 해상도의 순수 오디오 파일로 변환 및 저장 (ex: `answer_01.wav`)
3.  **STT Transcription:** 변환된 오디오 파일을 Whisper API에 전송하여 텍스트 추출
4.  **Output:** 변환된 텍스트 데이터를 JSON 형태로 패키징하여 면접관 화면(프론트엔드) 및 평가 리포트 분석 엔진으로 전달
