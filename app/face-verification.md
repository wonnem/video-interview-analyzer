# 👁️ AI 영어 면접: 지원자 본인 확인 및 신분증 마스킹 파이프라인

## 1. 기능 개요
지원자가 제출한 **이력서 사진**, 현장에서 촬영한 **신분증(주민등록증/운전면허증) 사진**, 그리고 면접 응시 중인 **실시간 웹캠 화면**의 얼굴을 대조합니다.
특히, 신분증 원본 이미지가 서버에 저장되거나 안면 인식 모델에 들어가기 전에 **주민등록번호 뒷자리 등 민감 정보를 자동으로 가려주는(Masking) 보안 전처리**를 일차적으로 수행합니다.

## 2. 사용 기술 스택 (MVP 2주 스프린트용)
* **EasyOCR (또는 Tesseract):** 신분증 이미지에서 글자가 있는 위치(Bounding Box)와 텍스트를 빠르게 추출.
* **OpenCV (`cv2`):** 추출된 주민등록번호 좌표를 바탕으로 해당 영역을 검은색 박스로 덮어버림(Masking) 및 얼굴 영역 크롭.
* **DeepFace (Python):** 마스킹이 완료된 안전한 이미지에서 얼굴 특징(Embedding)만 추출하여 동일인 여부 대조.

## 3. 데이터 처리 흐름도 (Data Flow)

**Step 1: 데이터 수집 (Input)**
프론트엔드로부터 `img_resume`(이력서), `img_id_card`(신분증 원본), `img_webcam`(실시간 프레임)을 수신합니다. (이때 데이터는 메모리에만 올려두고 디스크에는 즉시 저장하지 않습니다.)

**Step 2: 신분증 민감정보 마스킹 (PII Masking) ⭐핵심**
1. `img_id_card` 이미지를 EasyOCR에 통과시켜 텍스트와 좌표를 추출합니다.
2. 정규표현식(Regex)을 사용해 `\d{6}-\d{7}` 형태(주민번호 형식)를 띠는 텍스트를 찾습니다.
3. 해당 텍스트의 뒷자리 7개에 해당하는 좌표(또는 전체 번호 영역)를 OpenCV `cv2.rectangle` 함수를 이용해 검은색으로 칠해버립니다.
4. 처리된 '안전한 마스킹 신분증 이미지'만 다음 단계로 넘깁니다.

**Step 3: 얼굴 검출 (Face Detection)**
마스킹된 신분증, 이력서, 웹캠 이미지 각각에서 배경을 날리고 순수 '얼굴 영역'만 찾아내어 크롭 및 정렬합니다.

**Step 4: 안면 특징 추출 및 대조 (Face Verification)**
크롭된 3개의 얼굴 이미지를 DeepFace 모델에 통과시켜 벡터 값으로 변환한 뒤, 유사도를 계산합니다.
* **검증 A:** 이력서 사진 ↔ 마스킹 신분증 사진 얼굴 대조
* **검증 B:** 마스킹 신분증 사진 얼굴 ↔ 실시간 웹캠 대조

**Step 5: 결과 반환 및 데이터 즉시 파기 (Secure Output)**
대조 결과를 프론트엔드로 반환한 직후, 서버 메모리에 올라와 있던 원본 신분증 데이터는 복구 불가능하게 즉시 파기(Drop)합니다.

```json
{
  "applicant_id": "user_123",
  "verification_status": "PASS",
  "masked_id_url": "https://...", // 채용담당자가 확인할 수 있도록 마스킹 처리된 신분증만 임시 저장 후 경로 반환
  "match_scores": {
    "resume_vs_id": 0.89,
    "id_vs_webcam": 0.92
  }
}
